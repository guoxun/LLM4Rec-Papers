# OneMall: One Model, More Scenarios — End-to-End Generative Recommendation for E-commerce

**论文链接**: [arXiv:2601.21770](https://arxiv.org/abs/2601.21770)  
**机构**: 快手 (KuaiShou)  
**时间**: 2026年1月

---

## 核心问题：电商推荐的多场景统一

电商推荐场景的特殊性：

1. **多场景异构**：商品卡、短视频、直播间三大电商形态，各有不同的用户交互逻辑；
2. **传统方案各自为政**：每个场景维护独立的多阶段级联架构，模型割裂，难以共享知识；
3. **电商语义复杂**：商品的文本描述、价格、类目、品牌等多维属性需要统一建模。

---

## 核心方案：OneMall

**一个模型，覆盖更多场景**——将快手电商三大场景的推荐统一为单一端到端生成式框架。

### 电商语义分词器（三阶段构建）

OneMall 的核心创新之一是专为电商设计的 Semantic ID 方案：

```
LLM 微调（理解商品多维属性）
    ↓
Residual K-Means（Res-Kmeans）量化
    ↓
FSQ（Finite Scalar Quantization）精细化
```

- **LLM 微调**：利用大语言模型深度理解商品标题、价格、类目、品牌等结构化+非结构化信息；
- **Res-Kmeans**：多级残差量化，生成层级语义 ID；
- **FSQ**：有限标量量化，进一步精细化物品表示。

### 统一 Encoder-Decoder 生成架构

- Encoder 编码用户在三大场景的跨场景行为序列；
- Decoder 以自回归方式生成目标场景的推荐物品 ID；
- 通过**场景标识符（scenario token）**区分不同场景的生成目标。

### 多场景知识共享

- 用户在商品卡、短视频、直播间的行为信号**互相增强**；
- 统一模型捕捉跨场景的用户兴趣迁移规律；
- 避免每个场景独立维护模型的工程成本。

---

## 在线实验结果（快手电商）

| 场景 | 关键指标提升 |
|------|-------------|
| 商品卡 | 购买率显著提升 |
| 短视频电商 | GMV 提升 |
| 直播间 | 转化率提升 |

> OneMall 延续了 OneRec 在本地生活场景中 GMV +21% 的成功经验，进一步扩展到整个电商生态。

## 与 OneRec 的关系

```
OneRec（短视频推荐场景）
    ↓ 场景扩展
OneLoc（本地生活/地理位置场景）
    ↓ 电商多场景
OneMall（商品卡 + 短视频 + 直播间 电商统一）
```

OneMall 是 One系列"一个模型统一更多场景"核心理念在电商领域的落地。
